About Vocal

What is Vocal?

Vocal manages netCDF data product standards and the specifications that describe them. Data products are defined as pydantic models, and Vocal maps netCDF files onto those models so the full power of pydantic validation can be used to check compliance. It is geared towards datasets following the Climate & Forecast Conventions, but works with non-CF datasets too.

Origins

Vocal began life inside the core FAAM data processing pipeline, as the layer responsible for asserting that output files matched the agreed data product standards. As other groups and projects needed to validate netCDF data against their own conventions, it was extracted into a standalone package so the same machinery could be re-used outside of the original pipeline.

Today, Vocal is a general-purpose tool for defining, publishing, and checking netCDF data product standards — usable wherever a clear, versionable contract for scientific data is needed.
